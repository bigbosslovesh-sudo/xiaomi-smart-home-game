## 八、代码审查发现的问题

### BUG

1. **CinematicTransition 不会在关卡切换时重置状态** — 组件用 useState 初始化 stage='black'，但如果 React 复用了组件实例（同一个 key），state 不会重置。需要给 CinematicTransition 加 `key={currentLevel}` 确保每次关卡切换都重新挂载。

2. **第3关 reset 函数中 phase 设为 'play'** — 但第3关是 multiRule 模式，reset 后 ruleStates 会重新初始化，这是对的。但如果用户在 link 阶段点重置，应该回到 play 而不是 cards。当前逻辑：`setPhase(isCardMode ? 'cards' : 'play')` — 对第3关是正确的（不是 cardMode）。OK。

3. **设备重复放置** — 如果同一个设备名出现在多个房间的 placedDevices 里，`Object.values(placedDevices).flat().includes(dev.name)` 只检查名字是否存在，不检查是否已经放在正确房间。但由于每个设备只有一个 targetRoom，放对了就不会再拖，这个逻辑是对的。

4. **GLB 模型 Sketchfab transform 问题** — DoudouModel 组件手动重置了 gltf.scene 的 transform，但这修改了原始 scene 对象。如果组件重新渲染，useGLTF 返回的是缓存的 scene，已经被修改过了。这可能导致问题。应该 clone 后再修改。

### 需要修复

1. CinematicTransition 加 key
2. DoudouModel clone scene 后再修改 transform
